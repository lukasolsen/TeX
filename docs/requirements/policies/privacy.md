# Privacy

Policy revision: 2026-07-16

TeX is local-first. It does not require an account and does not include
telemetry, analytics, advertising, cloud storage, document upload, collaboration,
or AI/network features.

Project source, generated PDFs, build logs, recovery drafts, recent-project
paths, preferences, and workspace state remain on the device. TeX reads and
writes project files only in response to project workflows and stores app-owned
state in the operating system's application-data location. It invokes locally
installed TeX and SyncTeX commands for builds and navigation; those processes
inherit access allowed by the operating system and may read files referenced by
the project.

TeX itself does not transmit document content or usage data. Operating-system
services, a user-installed TeX distribution, packages, fonts, custom commands,
or future features may have their own behavior. No feature that sends project
data off-device may be introduced without an explicit product decision, a
specific consent design, and an update to this statement.

Removing a recent-project entry only removes TeX's reference to it. It does not
delete the project. Conversely, the current in-project delete command is
permanent; see [known limitations](known-limitations.md).
